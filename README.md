# 💻 [2026-01] 나만의 C언어 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
* Text RPG에서의 턴제 전투와 성장 시스템 구현

<br>


## 2. 시나리오 (5~10줄)
* 이 프로그램은 Text RPG에서의 전투 시스템과 전투 종료 후 획득하는 아이템을 통한 성장을 구현하는 프로젝트입니다.
* 프로그램 실행 시 플레이어는 원하는 직업을 선택(마법사, 전사, 도적 등)하면 그에 맞게 특화된 능력치을 가진 캐릭터가 생성되고 전투가 시작됩니다. 플레이어는 전투 단계에서 공격, 방어, 회복 3가지를 선택할 수 있고, 몬스터와 턴을 번갈아가며 전투합니다. 최종적으로 몬스터의 체력이 0이 되면 승리하고, 플레이어의 체력이 0이 되면 패배합니다. 플레이어 승리 시 아이템 보상을 획득하고, 아이템의 스탯에 기반하여 따라 플레이어의 스탯이 증가하는 구조를 가집니다. 이후에는 증가한 스탯을 가지고 더 강해진 적과 싸우며, 플레이어가 패배할 떄까지 전투를 진행하여 처치한 몬스터의 수로 점수를 나누는 시스템을 구현하는 것이 목적입니다. 간단하게 정리하자면 Text RPG의 턴제 전투 시스템을 구현하고, 아이템을 획득함으로써 증가하는 스탯과 플레이어가 패배할 때까지 이어지는 전투를 완성할 계획입니다.
 

<br>


## 3. 예상 기능 및 메뉴 (최소 5개)
1. 기본 능력치 생성(체력, 공격력, 방어력, 마나)

2. 플레이어의 직업 선택 (선택지 부여)

3. 직업 선택 시 직업에 따른 특화 능력치 부여(전사 : 높은 체력, 마법사 : 높은 마나. 도적 : 높은 공격력 등)

4. 선택 완료 후 전투 진입(전투 진입 문구 출력, 플레이어와 몬스터의 현재 상태 표시)

5. 플레이어 턴 시작(공격, 방어, 회복 선택지 부여)

6. 턴 종료 후 몬스터 턴 시작 (공격, 회복 중 랜덤으로 시전)

7. 4~5 과정 반복(어느 한 쪽의 HP가 0이 될 때까지, 음수는 불가능)

8. 몬스터의 체력이 0이 될 경우 승리 문구 출력, 플레이어의 체력이 0이 될 경우 패배 문구 출력

9. 승리 시 아이템 제공(위 작성한 4가지 능력치 중 하나 증가)

10. 아이템 수령 후 더 강한 몬스터와의 전투 시작 -반복

# 🚀 [버전별 개발 일지 & AI 협업 기록]

## 🟦 [1차 과제: V1.0] 기초 뼈대 구축 (입출력과 변수)
### **✨V1.0 개발 내용:**
  * 프로그램 실행 시 게임의 타이틀이 출력됩니다.
  * 플레이어가 원하는 문자 (문자열은 배열을 사용해야해서 제외했습니다) 입력 시 캐릭터가 생성 문구와 기본 능력치값(플레이어 이름 = 문자열, HP, MP = 정수, 공격력, 방어력 = 실수)이 출력됩니다.
  * start  입력 시 전투가 시작된다는 문구, 플레이어와 몬스터의 초기 상태를 출력합니다.
  * 플레이어가 몬스터를 공격합니다(플레이어의 공격력만큼 몬스터의 HP 차감).
  * 플레이어 -> 몬스터로 턴을 옮깁니다.(반복문을 사용하지 못해 1턴만 돌아갑니다.)
  * 몬스터가 플레이어를 공격합니다(플레이어가 받는 데미지 = 몬스터의 공격력*(몬스터의 공격력*플레이어의 방어력))
  * 플레이어와 몬스터의 공격으로 차감된 HP를 반영한 능력치를 화면에 출력합니다.
    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * C언어 4단계 과제에서 기본입출력 ➔ 조건문 ➔ 반복문 ➔ 배열/함수 과정을 만족하는 텍스트 RPG 코딩 시나리오와 기능확장 로드맵의 기획을 요청함.
    * **적용 내용:**
    * 1단계 : 플레이어의 능력치를 변수로 선언, 게임 타이틀 출력, 플레이어 이름 입력, 산술 연산을 이용한 전투 시 HP 감소 시스템 적용
    * 2단계 : 조건문을 이용하여 직업 선택지 부여, 턴 선택지 부여, 승패 및 아이템 판정 적용
    * 3단계 : 반복문을 이용하여 턴제 전투 루프, 몬스터의 인공지능(랜덤으로 공격과 회복 선택), 전투 종료 후 더 강한 적과의 전투 시스템 구현
    * 4단계 : 플레이어 이름 입력, 코드의 최적화 및 모듈화

 2. **내용 2**
    * **프롬프트 요약:**
    *1.  프로그램 실행 시 게임의 타이틀이 출력됩니다.
    *2. 플레이어가 원하는 닉네임 입력 시 캐릭터가 생성 문구와 기본 능력치값(플레이어 이름 = 문자열, HP, MP = 정수, 공격력, 방어력 = 실수)이 출력됩니다.
    *3. start  입력 시 전투가 시작된다는 문구, 플레이어와 몬스터의 초기 상태를 출력합니다.
    *4. 플레이어가 몬스터를 공격합니다(플레이어의 공격력만큼 몬스터의 HP 차감).
    *5. 플레이어 -> 몬스터로 턴을 옮깁니다.
    *6. 몬스터가 플레이어를 공격합니다(플레이어가 받는 데미지 = 몬스터의 공격력*(몬스터의 공격력*플레이어의 방어력))
    *7. 플레이어와 몬스터의 공격으로 차감된 HP, 소모된 마나 등을 반영한 능력치를 화면에 출력합니다.
    *위 내용과 1차 과제 조건(의미 있는 변수 5개 이상, 자료형 3개 이상(int, float/double, char 등) 혼합, 입출력 함수 사용(printf, scanf), 산술연산 1개 이상        사용)을 만족하고, 조건문, 반복문, 배열과 함수를 사용하지 않는 코드의 작성을 요청함.
      
    * **적용 내용:** 조건문과 반복문을 이용해야하는 선택시 시스템과 턴제 전투 루프 등을 배제하고, 게임 타이틀 출력(printf), 플레이어 이름 입력(scanf, 문자 자료형), 기본 능력치 부여(변수 선언, 자료형 사용/ 플), 플레이어와 몬스터의 HP 감소와 방어력에 따른 피해량 감소(산술연산자 사용)를 적용한 코드를 받음.
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** 플레이어 이름 입력 직후 모든 과정이 한 번에 진행되어 각 과정을 확인하지 못하는 문제가 발생함.
     * **해결:** AI에게 질문하여 'system("pause")' 를 이용해 각 과정을 하나씩 천천히, 그리고 선택적으로 볼 수 있도록 개선
  2. **문제 2:** 플레이어 이름을 입력하고자 했지만 배열(4단계) 없이는 문자를 하나만 입력할 수 있음.
     * **해결:** 플레이어 이름 입력을 1단계에서 4단계로 올려 1단계에서는 문자 입력을 통해 플레이어의 이름을 결정할 수 있게 코드를 구성함. 차후 함수와 배열을 배운 뒤에 플레이어 이름을 작성할 수 있도록 수정할 예정. 
  3. **문제 3:** 플레이어가 몬스터를 타격할 시 정수형으로 선언된 HP에서 실수형으로 선언된 공격력만큼을 빼야 하는 문제가 발생함.
     * **해결:** AI 질문과 구글링을 통해 실수를 정수형 HP에서 빼기 위해 (int)로 형변환하여 해결함.
